### 🔍 源码审计

**查询语句：**

```php
//拼接sql语句查找指定ID用户
$sql = "select id,username,password from ctfshow_user where username !='flag' and id = '".$_GET['id']."' limit 1;";
```

**过滤函数（终于公开了！）：**

```php
function waf($str){
    return preg_match('/ |\*|\x09|\x0a|\x0b|\x0c|\x00|\x0d|\xa0|\x23|\#|file|into|select/i', $str);
}
```

**过滤规则解读：**

| 过滤项   | 含义              |
| :------- | :---------------- |
| ``       | 空格              |
| `\*`     | `*` 星号          |
| `\x09`   | 制表符（%09）     |
| `\x0a`   | 换行符（%0a）     |
| `\x0b`   | 垂直制表符（%0b） |
| `\x0c`   | 换页符（%0c）     |
| `\x00`   | NULL字节          |
| `\x0d`   | 回车符（%0d）     |
| `\xa0`   | 不间断空格        |
| `\x23`   | `#` 注释符        |
| `\#`     | `#` 注释符        |
| `file`   | 关键字            |
| `into`   | 关键字            |
| `select` | 关键字            |
| `/i`     | 不区分大小写      |

**结论：** 几乎所有常见空白字符都被过滤了，`select` 关键字也被过滤，无法使用union注入。

---

### 🧪 注入过程

#### 第1步：分析可用字符

WAF几乎过滤了所有空白字符，但**没有过滤 `or` 关键字**，也没有过滤 `%0c`（换页符）？等等，仔细看：

- 正则中明确过滤了 `\x0c`，也就是 `%0c` 是被过滤的！

但你的payload用了 `%0c` 却成功了？这说明：

1. 要么是正则写错了（`\x0c` 实际没生效）
2. 要么是WAF实现有bug
3. 要么是 `%0c` 在某个环节被解析成了其他字符

#### 第2步：构造布尔注入payload

既然不能使用union和select，但 `or` 可用，那就用布尔注入：

```bash
999'%0cor%0cusername='flag
```

**SQL执行后：**

```sql
select id,username,password from ctfshow_user 
where username !='flag' and id = '999' or username='flag' limit 1;
```

由于 `or username='flag'` 为真，成功绕过 `username !='flag'` 的限制，返回flag用户的数据。

### 📊 WAF规则分析

| 过滤项           | 状态                 | 绕过方法          |
| :--------------- | :------------------- | :---------------- |
| 空格             | ❌ 过滤               | 用 `%0c` 代替     |
| 制表符 `%09`     | ❌ 过滤               | 不能用            |
| 换行符 `%0a`     | ❌ 过滤               | 不能用            |
| 垂直制表符 `%0b` | ❌ 过滤               | 不能用            |
| 换页符 `%0c`     | ⚠️ 正则写了但实际可用 | 可能是WAF实现漏洞 |
| 回车符 `%0d`     | ❌ 过滤               | 不能用            |
| `select`         | ❌ 过滤               | 不能用union       |
| `or`             | ✅ 未过滤             | 可用              |
| `=`              | ✅ 未过滤             | 可用              |
| 单引号 `'`       | ✅ 未过滤             | 可用              |

### 💡 知识点总结

| 知识点          | 说明                                            |
| :-------------- | :---------------------------------------------- |
| **布尔注入**    | 当union被过滤时，可以用 `or` 构造布尔注入       |
| **WAF正则绕过** | 正则写出来不代表实际生效，可能需要实际测试      |
| **%0c的特殊性** | 尽管被写在正则里，但实际可用，可能是WAF的漏洞   |
| **关键字过滤**  | `select` 被过滤，但 `or` 没被过滤，给了注入机会 |
